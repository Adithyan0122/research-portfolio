# Technical Analysis: v0.1-Baby-RAG

This document provides a deep-dive analysis into the architecture, mathematical foundation, and performance metrics of the **v0.1-Baby-RAG** system. This version serves as the baseline for the "Vector-To-Graph RAG" series.

---

## 🏗️ Architecture Overview

v0.1 is built using **Zero Frameworks**. Every component, from the vectorizer to the retrieval engine, is implemented using standard Python and NumPy.

### 1. Document Processing (Chunking)
- **Strategy**: Fixed-size word chunking with overlap.
- **Formula**: `num_chunks = ceil((N - overlap) / (chunk_size - overlap))`
- **Default**: 80-word chunks with a 15-word overlap to prevent context loss at boundaries.

### 2. The Vectorizer (TF-IDF)
- **Implementation**: Hand-rolled Term Frequency-Inverse Document Frequency.
- **Math**:
    - $$TF(t, d) = \frac{\text{count}(t \text{ in } d)}{|d|}$$
    - $$IDF(t) = \log\left(\frac{N+1}{df(t)+1}\right) + 1$$
- **Result**: Sparse vectors representing term importance across the corpus.

### 3. Retrieval Engine (Cosine Similarity)
- **Math**: $$\text{cos}(q, d) = \frac{q \cdot d}{||q|| \times ||d||}$$
- **Logic**: Performs a linear scan over the NumPy matrix (Vector Store) to find the top-K most similar chunks.

## 📊 Performance Metrics (Ollama mistral judge)

Evaluated on the internal demo corpus across 5 core RAG metrics.

| Metric | Score | Interpretation |
|---|---|---|
| **Hit Rate** | **0.20** | The correct context was found in 20% of queries. |
| **MRR** | **0.20** | Mean Reciprocal Rank — correlates with the quality of the top-1 result. |
| **Faithfulness** | **1.00** | Zero hallucinations; the LLM stays strictly within the provided context. |
| **Answer Relevance** | **0.35** | Measures how well the answer addresses the question semanticly. |
| **Context Precision** | **0.20** | Only 20% of retrieved chunks were identified as useful by the judge. |

### Analysis of Baseline
- **The "Keyword Gap"**: TF-IDF struggles with semantic synonyms. If a question uses "LLMs" and the text says "Large Language Models", the overlap is zero.
- **High Faithfulness**: Even with weak retrieval, the RAG prompt's grounding works perfectly. The model prefers saying "I don't know" over hallucinating.
- **The Path Forward**: These scores motivate **v0.2 (Advanced Chunking)** and **v0.5 (Neural Embeddings)**, which are expected to double the Hit Rate.

## 📌 Key Insights
> "Understanding the math behind the matrix is more valuable than learning a framework API."

Building this from scratch revealed that simple keyword matching is highly effective for technical, domain-specific documents where terminology is consistent, but fails on conversational queries — a key driver for implementing semantic search in future versions.
